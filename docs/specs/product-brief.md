# Greek Learning App — Product Brief
**Дата:** 2026-04-06
**Статус:** Утверждён перед разработкой

---

## О продукте

Telegram Mini App для изучения новогреческого языка с фокусом на подготовку к сертификационному экзамену A2 (ΠΙΣΤΟΠΟΙΗΣΗ ΕΛΛΗΝΟΜΑΘΕΙΑΣ). Продукт бесплатный — для себя, знакомых и всех желающих студентов.

**Платформа:** Telegram Mini App (React + Vite)
**Аудитория:** Самостоятельные учащиеся, готовящиеся к A2 или изучающие греческий с нуля
**Монетизация:** Отсутствует, бесплатно

---

## Архитектура

```
Telegram Client
    └── Mini App (React + Vite)
            └── FastAPI (Python)
                    └── PostgreSQL
```

Контент готовится заранее в **batch-режиме** и загружается в PostgreSQL до деплоя. Приложение только отдаёт готовый контент — нет зависимостей от внешних API в рантайме.

---

## Структура приложения

Два раздела:

| Раздел | Описание |
|--------|----------|
| **Учёба** | Юниты по темам и уровням A1 / A2 / B1 |
| **Словарь** | Карточки с повторением по интервалам (spaced repetition) |

Отдельного раздела "Экзамен" нет — экзаменационные форматы встроены в упражнения органически.

## Curriculum

### Уровни
```
A1 (6–8 юнитов)        ← стартовый уровень; пропустить можно только через placement test
  ├── Приветствия и знакомство
  ├── Числа, дни, время
  ├── Семья
  ├── Еда и напитки
  └── ...

A2 (8–10 юнитов)       ← на основе реальных экзаменационных билетов
  ├── Покупки и магазины
  ├── Транспорт и маршруты
  ├── Работа и профессии
  ├── Здоровье
  └── ...

B1 (8–10 юнитов)
  ├── Путешествия
  ├── Культура и традиции
  └── ...
```

### Состав юнита
- 15–20 карточек словаря
- 4–6 упражнений разных типов
- Мини-тест по итогам юнита

**Юнит считается завершённым**, когда пройдены все упражнения (типы 1–4, 6, 7) И сдан мини-тест. Тип 5 (карточки словаря) **не входит** в счётчик завершения — это вспомогательный элемент, доступный параллельно. Только после завершения начисляется XP +25 и карточки словаря юнита автоматически попадают в очередь повторений раздела "Словарь".

**Мини-тест:** 5 вопросов типов 1, 3, 4 — случайная выборка из лексики и тем юнита. Проходной балл не установлен — тест всегда можно завершить, результат фиксируется для статистики.

### Карта прогресса
- Все уровни (A1 / A2 / B1) видны на карте
- **Все юниты внутри уровня открыты сразу** — можно проходить в любом порядке
- По умолчанию пользователь начинает с A1
- Пропустить A1 и начать с A2 можно **только после прохождения теста на размещение (placement test)**

### Placement test (тест на размещение)
Аналог Duolingo: короткий тест при первом запуске, позволяющий пропустить уже знакомый уровень.

- Показывается **только при первом входе** в приложение
- ~15–20 вопросов из юнитов A1: лексика, базовые конструкции, типы 1/3/4
- Прошёл тест (≥ 80% правильных) → A1 помечается как "пропущен", открывается A2
- Не прошёл или пропустил тест → начинает с A1
- Пользователь может **пропустить тест** и всё равно начать с A1
- Пройти тест повторно нельзя (результат фиксируется), но A1 остаётся доступен для изучения в любой момент
- Если пользователь пропустил A1 через тест, но потом самостоятельно прошёл все его юниты — A1 **засчитывается как пройденный**, прогресс и XP начисляются в обычном порядке

## Типы упражнений (7 типов)

Все типы повторяют реальные форматы экзамена A2. Пользователь учит язык и одновременно привыкает к формату.

| # | Тип | Описание |
|---|-----|----------|
| 1 | **ΣΩΣΤΟ / ΛΑΘΟΣ** | Прочитай текст, отметь утверждения как верные или неверные |
| 2 | **Сопоставление** | Соедини начало фразы с окончанием (drag & drop или tap-to-match) |
| 3 | **Выбор из вариантов** | Выбери правильный ответ из 3–4 вариантов (α/β/γ/δ) |
| 4 | **Заполни пропуск** | Выбери слово из банка слов (слов чуть больше нужного) |
| 5 | **Карточки словаря** | Греческое слово ↔ перевод, кнопка "Слушать", алгоритм spaced repetition |
| 6 | **Περιγραφή εικόνας** | Прослушай и прочитай описание → выбери правильное изображение из сетки 6–8 |
| 7 | **Διάλογος** | Прочитай диалог → заполни таблицу из банка слов |

**Письменный ввод (Ερώτημα 5):** намеренно исключён — сложность автоматической проверки.
**Говорение:** намеренно исключено.

## Аудио

Аудио **обязательно** в приложении (прослушать слово или предложение). Говорение не предусмотрено.

**Генерация:** batch-скрипты через gTTS (Python, бесплатно), файлы MP3.
**Хранение:** Git LFS (бинарные файлы не попадают в основной git-объект). При деплое файлы копируются на сервер как обычная статика — пользователи скачивают их с сервера приложения, не из git.

**Где воспроизводится:**

| Тип | Поведение |
|-----|-----------|
| Тип 5 — Карточки словаря | Кнопка "Слушать" на каждой карточке |
| Типы 1, 3, 4 — Тексты упражнений | Кнопка "Слушать текст" |
| Тип 6 — Περιγραφή εικόνας | Аудио **воспроизводится автоматически** при первом показе задания + кнопка для повторного воспроизведения |
| Тип 7 — Диалог | Кнопка "Слушать" для каждой реплики |

## Spaced Repetition

**Алгоритм:** SM-2 с бинарным вводом ("знал" / "не знал")
- "Знал" → качество 4 → интервал растёт
- "Не знал" → качество 1 → сброс интервала

**Бейдж:** на разделе "Словарь" показывается количество карточек к повторению сегодня

## Геймификация и метрики

Принцип: интерактив есть, давления нет.

**Что есть:**

| Элемент | Описание | Хранение |
|---------|----------|----------|
| **Streak** | Дней подряд с активностью. Обновляется при **первом завершённом упражнении** за день (не при открытии). Обнуляется если пользователь не завершил ни одного упражнения за календарный день (по UTC) | `users.streak_days`, `users.last_active_date` |
| **Слов изучено** | Суммарное количество слов со статусом "learned" | `user_card_state.status` |
| **Прогресс юнита** | Сколько упражнений пройдено из N | `user_progress.exercises_completed` |
| **Прогресс уровня** | % завершённых юнитов A1 / A2 / B1 | Карта прогресса |
| **Точность сессии** | X / Y правильных ответов | Итоговый экран после упражнения |
| **Повторения сегодня** | Бейдж — сколько карточек ждут повторения | `next_review_at <= today` |
| **XP** | Накопительный счётчик за любую активность. Показывается на профиле как "Всего XP: 1 240". Без уровней, лиг и сравнения с другими | `users.total_xp` |

**Начисление XP:**

| Действие | XP |
|---|---|
| Завершил упражнение | +10 |
| Прошёл мини-тест юнита | +25 |
| Карточка "знал" | +2 |
| Поддержал streak (первое упражнение дня) | +5 бонус |

**Что отсутствует:** уровни, лиги, таймеры, жизни, штрафы, таблицы лидеров.

## Настройка: перевод заданий

Глобальный переключатель в настройках (не в каждом упражнении):

| Состояние | Отображение инструкции |
|-----------|----------------------|
| **Вкл** (по умолчанию) | Греческий + перевод: *"Σημειώνετε ΣΩΣΤΟ ή ΛΑΘΟΣ (отметьте верно/неверно)"* |

Упражнения внутри юнитов постепенно обучают читать инструкции на греческом.

## Offline и кэширование

Telegram Mini Apps работают в браузере — необходим offline-режим.

**Что кэшировать:** упражнения текущего юнита + изображения + аудио-файлы
**Стратегия:**
- Cache-first для статики (изображения, аудио, JS/CSS)
- Network-first для прогресса пользователя: при наличии сети — сохранение на сервер; при отсутствии — прогресс сохраняется локально (localStorage) и синхронизируется при следующем подключении
- Streak считается по **timezone пользователя** (если доступен из Telegram), а не по UTC — чтобы обнуление совпадало с ожиданиями пользователя

**Реализация:** Service Worker (Workbox или ручная настройка в Vite PWA plugin)

## Аутентификация

Пользователь идентифицируется через Telegram Mini App:
- При запуске Mini App Telegram передаёт `initData` с данными пользователя
- FastAPI верифицирует подпись `initData` через HMAC-SHA256 с bot token
- При первом запросе создаётся запись в таблице `users` (по `telegram_id`)
- Повторные запросы — поиск по `telegram_id`, запись уже существует

Дополнительная регистрация/логин не требуются.

## Генерация и модерация контента

**Генерация:** batch-скрипты на Python → PostgreSQL. Запускаются вручную по мере необходимости.

**Флоу одобрения:**
1. Скрипт создаёт контент с `is_published = false`
2. Валидация по чеклисту и JSON-схемам
3. Ручной просмотр через SQL / скрипт вывода
4. Установка `is_published = true` → контент появляется у пользователей

**Правило:** пользователи видят только контент с `is_published = true`.

Admin UI не нужен — избыточно для проекта данного масштаба.

## Ключевые сущности БД

| Сущность | Назначение |
|---|---|
| `users` | Telegram ID, streak, XP, дата последней активности |
| `curriculum_units` | Юниты по уровням A1/A2/B1 |
| `exercises` | Упражнения типов 1–7, флаг `is_published` |
| `vocabulary_cards` | Словарные карточки юнита с аудио |
| `user_progress` | Прогресс пользователя по юнитам |
| `user_card_state` | Состояние SM-2 на каждую карточку (интервал, дата повторения, статус) |

Детальная схема с полями — в архитектурном RFC.
