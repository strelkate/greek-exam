# PRD: Greek Learning App (Telegram Mini App)

**Дата:** 2026-04-06
**Статус:** Черновик
**На основе:** [Product Brief](product-brief.md) (утверждён 2026-04-06)
**Платформа:** Telegram Mini App (React + Vite / FastAPI / PostgreSQL)
**Монетизация:** Отсутствует

---

## 1. Цель продукта и проблема

### Проблема

Большинство ресурсов для изучения греческого языка либо дорогие, либо не ориентированы на конкретный сертификационный экзамен ΠΙΣΤΟΠΟΙΗΣΗ ΕΛΛΗΝΟΜΑΘΕΙΑΣ (A2). Самостоятельные учащиеся не имеют компактного инструмента, который:

- работает там, где они уже проводят время (Telegram);
- даёт практику в формате реального экзамена;
- не требует регистрации и оплаты;
- работает без стабильного интернета.

### Цель продукта

Создать бесплатный Telegram Mini App для изучения новогреческого языка уровней A1–B1 с основным фокусом на подготовку к экзамену A2. Приложение должно:

1. Давать системный курс (юниты по темам и уровням).
2. Повторять форматы реальных экзаменационных заданий.
3. Удерживать мотивацию через лёгкую геймификацию без давления.
4. Строить словарный запас через интервальные повторения (spaced repetition).

### Контекст

Проект одновременно решает образовательную задачу и служит портфолио-работой для фронтенд-разработчика. Это влияет на выбор технологий и объём Admin-инструментов (минимальный).

## 2. Целевая аудитория

### Персона 1 — Алинта, "Готовлюсь к экзамену"

- **Возраст:** 28 лет
- **Контекст:** Планирует сдать A2, изучает греческий самостоятельно 3–4 месяца
- **Где учится:** В мобильном телефоне, в перерывах и в транспорте
- **Проблема:** Не знает, какой формат будет на экзамене; боится, что учит "не то"
- **Ожидание от приложения:** Упражнения точно в формате A2, прогресс виден, учёба не занимает более 15–20 минут в день

### Персона 2 — Михаил, "Начинаю с нуля"

- **Возраст:** 42 года
- **Контекст:** Переезжает в Грецию, греческого не знает вообще
- **Где учится:** Дома вечером на телефоне, иногда в поездке без интернета
- **Проблема:** Не знает, с чего начать; нужен структурированный курс, а не набор случайных карточек
- **Ожидание от приложения:** Чёткая структура A1 → A2 → B1, offline-работа, минимум трений при старте

### Персона 3 — Кейт, "Поддерживаю уровень"

- **Возраст:** 35 лет
- **Контекст:** A1–A2 уже есть, хочет не терять навык и двигаться к B1
- **Где учится:** Неравномерно, 5–10 минут в разное время дня
- **Проблема:** Словарный запас деградирует без регулярной практики
- **Ожидание от приложения:** Раздел "Словарь" с повторениями по расписанию, streak как мотиватор

## 4. Функциональные требования

### 4.1 Аутентификация и онбординг

**FR-AUTH-01.** При каждом запуске Mini App FastAPI получает `initData` из Telegram, верифицирует HMAC-SHA256 подпись с bot token и извлекает `telegram_id`.

**FR-AUTH-02.** При первом запросе от нового `telegram_id` создаётся запись в таблице `users`. Последующие запросы находят существующую запись. Отдельный экран регистрации отсутствует.

**FR-AUTH-03.** При первом входе пользователю предлагается placement test. Пользователь может пропустить его нажатием кнопки "Пропустить".

**FR-AUTH-04.** Placement test: ~15–20 вопросов типов 1, 3, 4 из лексики и конструкций A1. Порог прохождения: ≥ 80% правильных ответов. При успехе A1 помечается как "пропущен" (`skipped`), A2 становится доступным как стартовый уровень. При неудаче или пропуске пользователь начинает с A1.

**FR-AUTH-05.** Placement test можно пройти только один раз. Результат фиксируется. A1 остаётся доступным для изучения в любом случае.

**FR-AUTH-06.** Если пользователь пропустил A1 через placement test, но затем вручную завершил все юниты A1, уровень A1 получает статус "завершён", прогресс и XP начисляются в обычном порядке.

### 4.2 Раздел "Учёба"

**FR-STUDY-01.** Главный экран раздела — карта прогресса с тремя уровнями: A1, A2, B1. Каждый уровень показывает процент завершённых юнитов.

**FR-STUDY-02.** Все три уровня видны на карте всегда. Все юниты внутри уровня открыты одновременно — пользователь проходит их в любом порядке.

**FR-STUDY-03.** По умолчанию (без placement test) пользователь начинает с A1; A2 и B1 видны, но отмечены как "ещё недоступны" до завершения предыдущего уровня. Исключение: после успешного placement test A2 сразу открыт.

> Примечание: точная логика блокировки уровней (открывать A2 только после всех юнитов A1 или нет) — уточнить перед реализацией. В брифе указано, что все юниты внутри уровня открыты; переход между уровнями явно не ограничен. Принимаем: уровни не блокируют друг друга, прогресс отображается информационно.

**FR-STUDY-04.** Экран юнита содержит: название, тему, список упражнений с отметкой выполнения, кнопку словаря юнита, кнопку мини-теста (доступна после завершения всех упражнений типов 1–4, 6, 7).

**FR-STUDY-05.** Юнит считается завершённым при выполнении обоих условий: (а) пройдены все упражнения типов 1–4, 6, 7; (б) сдан мини-тест юнита. Карточки словаря (тип 5) в счётчик завершения не входят.

**FR-STUDY-06.** После завершения юнита: начисляется +25 XP, карточки словаря юнита автоматически добавляются в очередь spaced repetition раздела "Словарь".

**FR-STUDY-07.** Каждый юнит содержит: 15–20 словарных карточек, 4–6 упражнений разных типов (не менее типов 1–4), мини-тест.

**FR-STUDY-08.** Мини-тест: 5 вопросов типов 1, 3, 4 — случайная выборка из лексики и тем юнита. Проходного балла нет — тест можно завершить с любым результатом. Результат фиксируется в статистике.

### 4.3 Типы упражнений

#### Тип 1 — ΣΩΣΤΟ / ΛΑΘΟΣ

**FR-EX1-01.** Показывается текст (2–6 предложений) на греческом. Под текстом — список из 3–5 утверждений.

**FR-EX1-02.** Пользователь отмечает каждое утверждение как ΣΩΣΤΟ (верно) или ΛΑΘΟΣ (неверно).

**FR-EX1-03.** Присутствует кнопка "Слушать текст" — воспроизводит аудио MP3 текста.

**FR-EX1-04.** После отправки ответов каждое утверждение подсвечивается зелёным (верно) или красным (неверно) с правильным ответом.

#### Тип 2 — Сопоставление

**FR-EX2-01.** Показываются два столбца: начала фраз (3–5 элементов) и окончания (то же количество).

**FR-EX2-02.** Взаимодействие: tap-to-match (нажать элемент в левом столбце, затем — в правом) или drag & drop. Реализация выбирается на этапе UI-дизайна.

**FR-EX2-03.** После отправки ответов показываются правильные пары; неверные пары подсвечиваются красным.

**FR-EX2-04.** Аудио не используется в данном типе.

#### Тип 3 — Выбор из вариантов

**FR-EX3-01.** Показывается вопрос или незаконченное предложение; 3–4 варианта ответа, помеченные α / β / γ / δ.

**FR-EX3-02.** Пользователь выбирает один вариант. После выбора отображается результат: зелёный — верно, красный — неверно с указанием правильного ответа.

**FR-EX3-03.** Присутствует кнопка "Слушать текст" — опциональное аудио для вопроса или стимула.

#### Тип 4 — Заполни пропуск

**FR-EX4-01.** Показывается предложение или короткий текст с пропусками. Под текстом — банк слов (слов на 1–2 больше, чем пропусков).

**FR-EX4-02.** Пользователь перетаскивает или нажимает слово из банка и помещает его в пропуск (tap-to-fill или drag & drop).

**FR-EX4-03.** После заполнения всех пропусков и отправки каждый пропуск подсвечивается зелёным или красным.

**FR-EX4-04.** Присутствует кнопка "Слушать текст".

#### Тип 5 — Карточки словаря

**FR-EX5-01.** Карточка показывает греческое слово на лицевой стороне. Пользователь флипает карточку (тап) для просмотра перевода.

**FR-EX5-02.** На каждой карточке присутствует кнопка "Слушать" — воспроизводит произношение слова.

**FR-EX5-03.** После просмотра перевода пользователь нажимает "Знал" или "Не знал". Алгоритм SM-2 обновляет интервал повторения.

**FR-EX5-04.** Тип 5 доступен внутри юнита как вспомогательный режим. В счётчик завершения юнита не входит.

#### Тип 6 — Περιγραφή εικόνας (описание изображения)

**FR-EX6-01.** При открытии задания аудио-описание воспроизводится автоматически. Присутствует кнопка повторного воспроизведения.

**FR-EX6-02.** Показывается сетка из 6–8 изображений. Пользователь выбирает одно изображение, соответствующее описанию.

**FR-EX6-03.** После выбора отображается результат: выделение правильного изображения зелёным, неверного — красным.

**FR-EX6-04.** Изображения хранятся локально в `public/images/exercises/` (SVG/PNG). Внешние URL не используются (требуется offline).

#### Тип 7 — Διάλογος (диалог)

**FR-EX7-01.** Показывается диалог (2–4 реплики). Рядом с каждой репликой — кнопка "Слушать".

**FR-EX7-02.** Под диалогом — таблица с пропусками и банк слов для заполнения.

**FR-EX7-03.** Механика заполнения и подсветка результата аналогичны типу 4.

### 4.4 Раздел "Словарь"

**FR-VOCAB-01.** Раздел показывает все карточки, добавленные из завершённых юнитов. Карточки, ещё не попавшие в очередь (юнит не завершён), в разделе не отображаются.

**FR-VOCAB-02.** Бейдж на разделе "Словарь" показывает количество карточек, у которых `next_review_at <= today`. Если таких карточек нет, бейдж не отображается.

**FR-VOCAB-03.** Сессия повторения включает только карточки, запланированные на текущий день. Карточки показываются по одной; пользователь оценивает знание бинарным выбором.

**FR-VOCAB-04.** Алгоритм SM-2 с бинарным вводом: "Знал" = качество 4 (интервал растёт), "Не знал" = качество 1 (интервал сбрасывается до 1 дня).

**FR-VOCAB-05.** Метрика "Слов изучено" = суммарное количество карточек со статусом `learned` (карточка достигла интервала ≥ 7 дней или иной порог — уточняется на этапе архитектуры).

**FR-VOCAB-06.** Карточка всегда воспроизводит аудио по нажатию "Слушать" независимо от стороны (лицевая / обратная).

### 4.5 Геймификация

**FR-GAME-01. Streak.** Хранится в `users.streak_days`. Обновляется при первом завершённом упражнении за день. Обнуляется, если за полный календарный день (по timezone пользователя, если доступен; иначе — UTC) не завершено ни одно упражнение.

**FR-GAME-02. XP.** Накопительный счётчик `users.total_xp`. Начисление:

| Действие | XP |
|---|---|
| Завершил упражнение (тип 1–4, 6, 7) | +10 |
| Прошёл мини-тест юнита | +25 (в составе +25 за завершение юнита) |
| Карточка "Знал" (тип 5 или раздел "Словарь") | +2 |
| Поддержал streak (первое упражнение дня) | +5 бонус |

**FR-GAME-03.** XP отображается на экране профиля как "Всего XP: [число]". Уровни, лиги, сравнение с другими пользователями отсутствуют.

**FR-GAME-04.** Streak отображается на главном экране или в шапке приложения.

**FR-GAME-05.** Точность сессии (X / Y правильных ответов) показывается на итоговом экране после каждого упражнения.

---

### 4.6 Настройки

**FR-SETTINGS-01.** Глобальный переключатель "Перевод инструкций": включён по умолчанию. При включении рядом с греческой инструкцией показывается её перевод (например, "Σημειώνετε ΣΩΣΤΟ ή ΛΑΘΟΣ — отметьте верно/неверно"). При отключении — только греческий текст.

**FR-SETTINGS-02.** Настройка применяется ко всем упражнениям немедленно, без перезагрузки.

---

### 4.7 Контент и модерация

**FR-CONTENT-01.** Пользователи видят только контент с флагом `is_published = true`.

**FR-CONTENT-02.** Контент генерируется batch-скриптами на Python до деплоя; рантаймовой зависимости от внешних API нет.

**FR-CONTENT-03.** Аудио-файлы генерируются gTTS в batch-режиме, хранятся как MP3 в Git LFS, при деплое копируются на сервер как статика.

**FR-CONTENT-04.** Изображения для типа 6 хранятся в `public/images/exercises/` (SVG/PNG). Поле `image_path` в БД содержит относительный путь.

## 5. Нефункциональные требования

### 5.1 Производительность

**NFR-PERF-01.** Первый contentful paint Mini App ≤ 2 секунд на 4G.

**NFR-PERF-02.** Переход между экранами (карта → юнит → упражнение) ≤ 300 мс при наличии кэша.

**NFR-PERF-03.** Аудио начинает воспроизводиться ≤ 500 мс после нажатия кнопки при наличии кэша.

**NFR-PERF-04.** API-ответы FastAPI на запросы контента (список упражнений, карточки юнита) ≤ 300 мс (p95) при нагрузке до 50 одновременных пользователей.

### 5.2 Offline

**NFR-OFFLINE-01.** Service Worker (Workbox или Vite PWA plugin) обеспечивает offline-доступ к контенту текущего открытого юнита: упражнения, изображения, аудио.

**NFR-OFFLINE-02.** Стратегия для статики (JS/CSS, изображения, аудио): **cache-first**. Обновление при следующем онлайн-запуске.

**NFR-OFFLINE-03.** Стратегия для прогресса пользователя: **network-first**. При отсутствии сети прогресс записывается в localStorage. При восстановлении соединения — синхронизация с сервером.

**NFR-OFFLINE-04.** Конфликты синхронизации (одновременное изменение прогресса offline и на сервере) разрешаются по принципу "выбрать наибольший прогресс" (more-complete wins). Детальная стратегия — в архитектурном RFC.

### 5.3 Безопасность

**NFR-SEC-01.** Каждый запрос к FastAPI содержит заголовок с `initData`. Сервер верифицирует HMAC-SHA256 подпись. Запросы без валидной подписи отклоняются с кодом 401.

**NFR-SEC-02.** Bot token не хранится на клиенте и не передаётся в ответах API.

**NFR-SEC-03.** Все взаимодействия с БД используют параметризованные запросы (ORM или явный binding); SQL-инъекции исключены.

**NFR-SEC-04.** CORS настроен только для доверенных доменов Telegram Mini App.

**NFR-SEC-05.** Пользователь не может изменить прогресс другого пользователя: все write-операции проверяют соответствие `telegram_id` из `initData` и `user_id` в запросе.

### 5.4 Надёжность и доступность

**NFR-REL-01.** Uptime цели: 99% в месяц (бесплатный проект, разумный минимум).

**NFR-REL-02.** Ошибки сети и API отображаются пользователю в виде понятного сообщения, а не пустого экрана.

**NFR-REL-03.** Прогресс сессии сохраняется при фоновом переходе из Mini App (Telegram может свернуть приложение).

### 5.5 Доступность (a11y)

**NFR-A11Y-01.** Интерактивные элементы имеют тапабельную область не менее 44×44 dp.

**NFR-A11Y-02.** Кнопки и иконки сопровождаются `aria-label` для screen reader.

**NFR-A11Y-03.** Контраст текста соответствует WCAG AA (4.5:1 для основного текста).

### 5.6 Совместимость

**NFR-COMPAT-01.** Telegram Mobile (iOS / Android) — основная платформа, поддержка обязательна.

**NFR-COMPAT-02.** Telegram Desktop / Web — вторичная платформа; критические сценарии должны работать, отдельная оптимизация не приоритетна.

**NFR-COMPAT-03.** Минимальная версия iOS: 15; Android: Chrome 90+.

## 6. Out of Scope

Следующие возможности явно исключены из текущей версии продукта:

| Что | Причина |
|-----|---------|
| Говорение и проверка произношения | Сложность реализации, выходит за рамки формата Mini App |
| Письменный ввод (свободный текст, Ερώτημα 5) | Сложность автоматической проверки |
| Таблицы лидеров, социальные сравнения | Противоречит принципу "без давления" |
| Жизни, таймеры, штрафы | Противоречит принципу "без давления" |
| Admin UI для управления контентом | Избыточно для масштаба проекта |
| Платёжный сценарий, монетизация | Продукт бесплатный |
| Веб-версия вне Telegram | Платформа — Telegram Mini App |
| Push-уведомления о streak | Технически возможно в Telegram, но не в текущем скоупе |
| Пользовательский профиль с аватаром | Telegram аватар отображается автоматически при необходимости |

## 7. Критерии успеха / Definition of Done

### 7.1 Definition of Done — отдельное упражнение

- [ ] Упражнение корректно отображается на iOS и Android Telegram
- [ ] Аудио-кнопки воспроизводят файл (или показывают ошибку, если файл недоступен)
- [ ] Правильные/неверные ответы подсвечиваются зелёным/красным после проверки
- [ ] Итоговый экран показывает точность (X / Y)
- [ ] XP начисляется при завершении упражнения
- [ ] Streak обновляется при первом упражнении дня
- [ ] Прогресс сохраняется в БД; при offline — в localStorage

### 7.2 Definition of Done — юнит

- [ ] Все упражнения типов 1–4, 6, 7 доступны и проходимы
- [ ] Словарные карточки (тип 5) доступны как вспомогательный режим
- [ ] Мини-тест открывается только после завершения всех обязательных упражнений
- [ ] После прохождения мини-теста: юнит помечается завершённым, +25 XP начислены, карточки добавлены в очередь "Словарь"
- [ ] Прогресс юнита отображается корректно на карте

### 7.3 Definition of Done — раздел "Словарь"

- [ ] Бейдж показывает корректное число карточек на сегодня
- [ ] Сессия повторения работает по алгоритму SM-2
- [ ] После сессии интервалы обновлены в `user_card_state`
- [ ] Метрика "Слов изучено" обновляется

### 7.4 Definition of Done — онбординг

- [ ] Placement test показывается только при первом входе
- [ ] Результат теста корректно открывает/не открывает A2
- [ ] Пропуск теста корректно отправляет пользователя в A1
- [ ] Повторное прохождение placement test невозможно

### 7.5 Definition of Done — offline

- [ ] Приложение запускается и показывает кэшированный контент при отключённой сети
- [ ] Прогресс сессии сохраняется локально и синхронизируется при восстановлении сети
- [ ] Аудио и изображения текущего юнита доступны offline

### 7.6 Метрики успеха продукта (наблюдательные, не технические)

Так как монетизация отсутствует, метрики носят качественно-наблюдательный характер:

| Метрика | Целевой ориентир |
|---------|-----------------|
| Пользователи завершили ≥ 1 юнит | Подтверждает, что флоу юнита работает end-to-end |
| Пользователи вернулись на 3+ день (streak ≥ 3) | Подтверждает базовое удержание |
| Ошибки 5xx в FastAPI | < 1% запросов |
| Средний размер бандла Mini App | ≤ 500 КБ gzipped (без аудио) |
